pwntest.modules.binary module#

The Binary Automation module is all about interacting with Linux ELF binaries. You can load other binary file formats into this module, but some of the functionality will not work.

BinaryAutomation is a class that provides some functionality for interacting with executables.

assert_exploit(exploit, flag='', flag_path='', **kwargs) → bool#

Assert an exploit drops a shell or returns a flag. A few important things

  • If the exploit drops a shell, the shell should be returned by the exploit function. This is the default if flag and flag_path are not provided.

  • If both flag and flag_path are provided, a tube should be returned by the exploit function and pwntest will check the flag is in flag_path

  • If flag is provided and not flag_path, pwntest will check the flag is in return value of the exploit function

Parameters:

  • exploit – a python function that either returns a shell or a flag string, as specified by params supplied to the function.

  • flag – Flag inside the flag file

  • flag_path – Path to the flag file

  • kwargs – User defined keyword arguments to pass to the exploit function

Returns:

True if the exploit works, False otherwise

Examples:

Remote Exploit

>>> def exploit(ip, port):
>>>     s = remote(ip, port)
>>>     s.sendline(b"cat /flag")
>>>     return s.recvline_contains(b"flag{")
>>> tester.BinaryAutomation.assert_exploit(exploit, flag="flag{", flag_path="/flag", ip="127.0.0.1", port=4444)

Local Exploit

>>> def exploit():
>>>     s = process("./demo")
>>>     s.sendline(b"FOOBAR")
>>>     # shell dropped, so s is a tube into the shell
>>>     return s
>>> tester.BinaryAutomation.assert_exploit(exploit=exploit_shell)
assert_protections(protections: list) → bool#

Check if a list of string protection names are present in the binary.

Options:

  • NX || NX Stack

  • Canary || Stack Canary || Stack Protector

  • PIE || Position Independent Executable

  • RELRO Full || Full RELRO

  • RELRO Partial || Partial RELRO

Parameters:

protections – List of protections to check for

Returns:

True if all protections are present, False otherwise.

Example:

>>> tester.BinaryAutomation.assert_protections(["NX", "Canary", "PIE", "RELRO Full"])
[!] Binary does not have stack canary.
False
assert_rop_gadget_exists(gadget: list, deep_search=False) → bool#

Check if a ROP gadget exists in the binary.

Parameters:

  • gadget – Gadget to check for. Same format as pwnlib.rop.rop.ROP.find_gadget()

  • deep_search – If True, do a more intense search for the gadget. This can return false positives.

Returns:

True if gadget exists, False otherwise.

Example:

>>> tester.BinaryAutomation.assert_rop_gadget_exists(["pop rdi", "ret"])
False

>>> tester.BinaryAutomation.assert_rop_gadget_exists(["ret"])
True
